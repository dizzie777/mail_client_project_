# ФУНКЦИОНАЛЬНОЕ ТЕСТИРОВАНИЕ

## Тест-кейс 1:

Основной сценарий работы пользователя

1. Загрузка приложения:
   Проверьте в консоли:

- Загружен ли config.js
- Проверен ли сервер API
- Загружены ли письма

2. Просмотр письма:
   Нажмите на любое письмо в списке. Проверьте:

- Справа отобразилось содержимое
- Непрочитанное письмо пометилось прочитанным
- В консоли нет ошибок

3. Создание нового письма:
   Действия:

1) Нажать "Новое письмо"
2) Заполнить форму
3) Нажать "Отправить"

Проверки:

- Форма открылась
- Валидация работает
- Письмо появилось в "Отправленных"
- Уведомление об успехе показалось

4. Ответ на письмо:

Выберите письмо, нажмите "Ответить"

- Проверьте автозаполнение полей
- Отправьте ответ

5. Удаление письма:
   Выберите письмо, нажмите "Удалить"

- Подтвердите действие
- Проверьте перемещение в корзину

## Тест-кейс 2:

Работа с папками

1. Переключайтесь между папками:

- Входящие
- Отправленные
- Черновики
- Корзина

2. Проверьте:

- Счетчики обновляются
- Содержимое меняется
- Активная папка подсвечивается

## Тест-кейс 3:

Поиск и фильтрация

1. В поле поиска введите:

- Существующий текст (должны найтись письма)
- Несуществующий текст (должна быть пустая страница)
- Очистите поиск (должны вернуться все письма)

## Тест-кейс 4:

Обработка ошибок

1. Отключите backend сервер и проверьте:

- Статус API стал "офлайн"
- Появилось уведомление об ошибке
- Приложение не "упало"

2. Введите некорректные данные в форму:

- Неправильный email
- Пустую тему
- Короткий текст
- Проверьте валидацию

   - Причина: Попытка получить письмо с несуществующим ID
   - Решение: Добавил проверку if (letter) перед обращением к свойствам

   - Причина: Неправильные заголовки ответа сервера
   - Решение: Настроил CORS в server.js
